# biz-leyu-resource

## 项目简介

`biz-leyu-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。该仓库不针对任何特定域名或网站，仅作为 HTML 静态页面的集中存放与版本管理使用。适合用于演示、存档、轻量页面分发等场景。

## 目录说明

```
biz-leyu-resource/
├── pages/          # 存放各个独立 HTML 页面（按目录或文件名组织）
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
└── index.html      # 可选：项目入口页面（可自定义跳转或列表）
```

- `pages/`：每个子文件夹或文件代表一个独立页面，便于分类管理。
- `assets/`：存放多个页面共用的样式、脚本、字体、图片等资源。
- `index.html`：默认首页，可用于展示页面索引或导航信息。

## 页面归档说明

- 所有 HTML 页面均为独立文件，不依赖外部数据库或后端服务。
- 页面之间相互独立，无强制关联，便于单独访问或引用。
- 归档时建议保持清晰的命名规范，便于后续查找和维护。
- 如需引入公共资源，请统一放置在 `assets/` 目录下，避免资源散落。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，确保路径无冲突。
- 更新页面：直接修改对应 HTML 文件，建议附带简要的 commit 备注说明变更内容。
- 删除页面：移除对应文件或目录，并检查是否有残留引用。
- 资源管理：公共资源更新后，注意检查各页面引用是否有效。
- 建议使用 Git 分支管理实验性或未完成的页面，避免影响主分支稳定性。
